# Watch Up — 통합 웹 기획 프롬프트

너는 React, Vite, Node.js, Express.js, Firebase, Firestore 실시간 데이터 처리, 금융 대시보드 UX를 20년 이상 개발해온 시니어 풀스택 개발자다.

나는 “Watch Up”라는 실시간 모의 투자 웹 서비스를 만들고 싶다.

목표:

* 사용자가 자신이 보유한 주식을 직접 추가
* 실시간 주가 데이터 반영
* 수익률 계산
* 종목별 차트 확인
* 총 자산 현황 확인
* 실제 투자 앱 같은 UX 제공

전체 프로젝트는 실제 서비스 수준의 구조로 설계해야 한다.

# 핵심 설계 원칙
---

1. 프론트엔드가 구동되는 각 클라이언트 환경(개인 IP)에서 Python 프로세스 또는 Web WebAssembly/로컬 스크립트를 통해 yfinance API를 직접 호출하여 개별 종목의 상세 데이터 및 초기 차트 데이터를 조회한다.
2. 모든 실시간 주식 데이터는 백엔드 서버를 통해 전달한다.개별 주식 조회 및 초기 로딩 데이터는 클라이언트(개인 IP)에서 직접 처리하되, 유저 간 공유가 필요하거나 백엔드 캐싱이 가미된 공통 실시간 마켓 데이터에 한해서만 백엔드 서버를 경유하도록 이원화한다.
3. API KEY는 서버에서만 관리한다.
4. 서버에서 캐싱 후 프론트에 전달한다.
5. 클라이언트(개인 IP) 측에서 주기적으로 데이터를 폴링(Polling)하거나 로컬에서 갱신하는 것을 기본으로 하되, 서버측 통합 인덱스나 실시간 알림 피드가 필요한 영역에만 제한적으로 Socket.io를 활용한다.
6. 모바일 반응형까지 고려한다.
7. 실제 금융 서비스 느낌의 UX를 구성한다.
8. 다크모드를 기본으로 고려한다.
9. 상승은 빨간색, 하락은 파란색으로 표시한다.
10. Skeleton UI를 적극 사용한다.

예시:

Express API 호출

↓

↓
yfinance 조회
↓
JSON 반환
↓

↓
Socket.io 실시간 전파

# Stock Chart
* Stock Chart 규칙
1. 그래프 우측에 들어갈 가격은 아래와 같이 적용한다.
  무조건 소수점은 버림한다.
  해당 기간의 최저점 가격은 그래프 최하단에 위치
  해당 기간의 고점 가격 + 20%를 그래프 최상단에 위치
  최하단과 최상단 사이에는 3개의 포인트가 있고 아래에서부터 (최저점) + 25%, + 50%, + 75%로 구성된다.
2. 그래프 하단에 들어가는 기간은 아래와 같이 표시하고 그래프를 반영한다.
  정한 일수: 그래프 좌측 끝, 그래프 중간, 그래프 우측 끝
  1 Day: 0 H 12 H 24 H, 
  1 Week: Mon Wed Fri, 
  1 Month: 1 D, 15 D (last day) D
  1 Year: Jan, Jun, Dec
  Whole period: (first year to go public) Y, (middle year) Y, (this year) Y
3. 그래프 내부 클릭시 해당 시간의 가격만 보여준다. 가격은 소수점 둘째자리까지 표현하고 나머지는 버림한다.
4. 그래프 외부 클릭시 내부에 보여지고 있는 가격이 있다면 보여주지않게 만든다. 없을 경우 아무일도 일어나지 않는다.
